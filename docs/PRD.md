# PRD - 会议桌牌自动排序工具

## 1. 产品定位
- **一句话描述**：输入主客双方人数，自动按座次礼仪规则生成可编辑的会议桌牌摆放 SVG 平面图。
- **目标用户**：公司行政部人员（非技术背景，日常高频使用）

## 2. 用户痛点
- **痛点1 — 效率低**：每次会议手工在 Excel/PPT 画平面图、计算座次顺序，耗时 20-40 分钟。（现有方案：Excel/PPT 手动画图）
- **痛点2 — 易出错**：以左为尊、单双数排列、面门为尊等规则复杂，人工计算易搞反左右。（现有方案：靠资深行政人员经验记忆）
- **痛点3 — 经验断层**：座次规则在资深人员脑子里，新人无法上手，人员离职即经验流失。（现有方案：口头传授 + 纸质模板）
- **痛点4 — 不可复用**：通用画图工具（Canva/ProcessOn）无座次规则引擎，每次需重新计算，模板也无法适应人数变化。

## 4. 功能黑名单（明确不做）
- **大型会议室（主席台 + 观众席前排领导）**：MVP 仅做长方形会议桌，后期迭代
- **用户登录 / 权限系统**：纯本地工具，无需账号
- **在线协作 / 多人编辑**：单人使用场景
- **手机端适配**：MVP 仅桌面浏览器
- **会议室预约 / 审批流**：属 OA 系统范畴
- **短边座位自动排序**：短边仅绘制占位，不参与自动排列

## 5. 页面清单
- **页面1 — 主界面（唯一页面）**：输入区（客方人数、主方人数）+ SVG 预览区 + 下载按钮

## 6. 核心流程
1. 用户双击打开 `index.html`
2. 输入客方人数（0–8）、主方人数（0–8）
3. 页面实时渲染 SVG 平面图，座位自动排列
4. 用户检查无误，点击「下载 SVG」
5. （可选）将 SVG 拖入 PPT 微调（如替换数字为真实姓名、调整短边座位）
6. 打印或发送给现场布置人员

## 7. 技术方案
| 维度 | 选型 | 理由 |
|------|------|------|
| 交付形态 | 单文件 HTML（`会议座次排列.html`） | 双击即用，零安装，U 盘拷贝即用 |
| 渲染方案 | 原生 SVG（JavaScript 拼接 SVG 字符串） | 矢量图，可编辑，可拖入 PPT/Word |
| 逻辑语言 | 纯 JavaScript（原生） | 浏览器直接跑，无需 Node.js / npm |
| 依赖 | 零依赖 | 无框架、无 CDN 外链、无构建工具 |
| 部署 | 无需部署 | 本地文件双击打开，也可放公司内网静态服务器 |
| 导出 | `Blob` + `<a download>` | 浏览器原生下载，生成 .svg 文件 |

### 会议室布局规格
```
                    短边 2座（仅占位，不参与排序）
            ┌──────────────────────────┐
            │  A7 A6 A5 A4 A3 A2 A1 A0 │  ← 长边A：客方 8座（上座，面门）
            │                          │
            │       [ 会 议 桌 ]       │
            │                          │
            │  B0 B1 B2 B3 B4 B5 B6 B7 │  ← 长边B：主方 8座（背靠门）
            └──────────────────────────┘
                    白板（无座）          门1  门2
```
- 长边 A：客方 8 座，面门为尊
- 长边 B：主方 8 座，门在此侧
- 短边顶部：2 座（仅绘制占位，用户自行在 SVG 中微调）
- 短边底部：电子白板，无座位
- 门位置：长边 B 一侧，共 2 个门

### 座次排列规则
- **以左为尊**：1 号居中，2 号在 1 号左手边，3 号在右手边，依次左右交替（注意：客方面门、主方面白板，双方面对面，"左手"方向在俯视图上相反）
- **居中为尊**：1 号（奇数）或 1 号+2 号（偶数）居中
- **面门为尊**：正对门的长边 A 为上座，安排客方
- **主客分明**：客方坐长边 A，主方坐长边 B
- **主客 1 号对坐**：1 号位由主方人数决定（主方优先原则）——主方 ≤7 人→1 号在第 4 位，主方 =8 人→1 号在第 5 位；客方 1 号对坐同列。双方各按"以左为尊"独立排列。双方均为偶数时，1、2 号各自居中，不要求正对。

## 8. 主要风险
- **风险1 — 座次规则理解偏差**：以左为尊的"左右"方向在平面图中的坐标映射可能出错。（应对：用已知案例验证输出，如客方 5 人、主方 5 人时，1 号位置是否正确）
- **风险2 — SVG 在 PPT 中的兼容性**：部分旧版 PPT 对 SVG 支持有限。（应对：测试 WPS/Office 365/Office 2016+ 的 SVG 导入；必要时提供 PNG 降级方案）
- **风险3 — 会议室布局与实际不符**：不同会议室布局可能略有差异。（应对：MVP 锁定一间标准会议室；远期支持多模板）
